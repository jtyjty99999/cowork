# 代码审查 Skill

对指定的代码进行全面审查，生成详细的审查报告。

## 审查维度

### 1. 代码质量
- 命名规范：变量、函数、类的命名是否清晰
- 代码结构：是否遵循单一职责原则
- 可读性：代码是否易于理解

### 2. 潜在问题
- 错误处理：是否有适当的异常处理
- 边界条件：是否考虑了边界情况
- 安全隐患：是否存在安全漏洞

### 3. 性能考量
- 时间复杂度：算法效率如何
- 空间复杂度：内存使用是否合理
- 潜在瓶颈：是否有性能瓶颈

### 4. 最佳实践
- 是否遵循语言/框架的最佳实践
- 是否有重复代码可以抽取
- 测试覆盖率建议

## 审查目标
$ARGUMENTS

## 输出格式

```markdown
# 代码审查报告

## 概述
[简要描述审查的代码范围和整体印象]

## 发现的问题
### 🔴 严重问题
- [问题描述和位置]

### 🟡 建议改进
- [改进建议]

### 🟢 优点
- [代码中做得好的地方]

## 具体建议
[详细的改进建议和代码示例]
```
